\subsection{Iterated Local Search}
The idea of iterated local search is that instead of searching the whole space of all candidate solutions, we focus only on candidate solutions in the local neighborhood of an initial solution.
These are usually generated by some kind of local search heuristic.
It is conceptually simple, but has led to state-of-the-art algorithms for many problems \cite{lourencoIteratedLocalSearch2019}.
In its essence, iterated local search builds a chain of solutions by exploring the currently best solutions local neighborhood but introduces some randomness in order to not get stuck in local minima.

\begin{algorithm}[H]
	\caption{Iterated Local Search}
	\label{alg:ils}
	\begin{algorithmic}[1]
		\STATE $s_0 = \text{GenerateInitialSolution}()$
		\STATE $s^* = \text{LocalSearch}(s_0)$
		\STATE $s_{\text{best}} = s^*$
		\REPEAT
		\STATE $s' = \text{Perturbation}(s^*)$
		\STATE $s^{*\prime} = \text{LocalSearch}(s')$
		\IF{$\text{Cost}(s^{*\prime}) < \text{Cost}(s_{\text{best}})$}
		\STATE $s_{\text{best}} = s^{*\prime}$
		\ENDIF
		\IF{$\text{Cost}(s^{*\prime}) < \text{Cost}(s^*)$}
		\STATE $s^* = s^{*\prime}$
		\ENDIF
		\UNTIL{termination condition met (e.g., $\text{iterations} = 200$)}
		\RETURN $s_{\text{best}}$
	\end{algorithmic}
\end{algorithm}
Iterated local search can be used with any heuristic optimization algorithm (\textit{LocalSearch}) that takes in a tour and returns one that is at least as good as the input.
It starts from some kind of initial solution \( s_{0} \) that is improved by one iteration of the \textit{LocalSearch} algorithm, this is the current solution \( s^* \).
Then we try to improve \( s^* \) for a number of iterations or until a termination condition is met.

For the algorithm to be able to escape the local neighborhood, instead of applying \textit{LocalSearch} directly to the current solution \( s^* \),
first a change or \textit{Perturbation} is applied that leads to an intermediate solution \( s' \).
Then \textit{LocalSearch} is applied to \( s' \) and we get a new candidate solution \( s^* ' \).
If this candidate solution \( s^* ' \) has a better \textit{Cost} than the current one, it becomes the new current solution \( s^* \).
If it is better than the current best, it also becomes the new current best \( s_{best} \).


The \textit{Cost} is the function that is to be minimized by the algorithm.
In the case of the TSP that is the length of the tour.

The \textit{Perturbation} is the mechanism that allows the algorithm to escape local minima. This function should not be reversible by the \textit{LocalSearch}.
So if the algorithm steps from \( s^*_1 \to s^*_2 \) it then should be unable to undo it \(  s_2^* \to  s_1^* \).

Usual choices, and what was also used in this study are 2-Opt for the \textit{LocalSearch} and 4-Opt for the \textit{Perturbation},
this is a good choice because a 4-Opt move cannot be undone by a 2-Opt move and thereby return to the last starting point of the current iteration.

4-Opt (also called double-bridge move) randomly splits the tour into four segments \( A+B+C+D \) and reconnects them as \( A+C+B+D \) \cite{lourencoIteratedLocalSearch2019}.
